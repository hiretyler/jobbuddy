The bookmarklet

Drag this to your bookmarks bar. On any job posting, click it to grab the JD and send it to your Inbox, scored. It reads same-origin embeds (iCIMS) and hands cross-origin board iframes (Greenhouse, Lever) to JobBuddy to fetch.

Relaunch from a fresh clone

If you ever start over from a GitHub download, the repo does not include secrets or personal data. You need .env, the Google/Gmail credentials, plus your contact info and career bank (kept local-only so they stay out of the public repo) before it will run.

  1. Install deps: npm install
  2. Restore config: drop in .env (keys: GOOGLE_SHEET_ID, the GMAIL_OAUTH_* set, GMAIL_TOKEN_PATH, CLAUDE_BIN, CLAUDE_MODEL, ARCHIVE_DIR, PORT), plus secrets/service-account.json and tokens/gmail.json.
  3. Restore personal data (gitignored, never in the repo): secrets/contact.json (resume contact info - copy assets/contact.example.json and fill it in) and secrets/master-bank.json (your career bank for ATS swaps - assets/master-bank.example.json shows the schema). Without these the app still runs but serves the placeholder contact and a near-empty swap bank.
  4. Check the claude CLI is installed and logged in (all model work runs through it - no API key).
  5. Repair the sheet headers (optional): npm run bootstrap-sheet
  6. Run it. Quick foreground: npm start. Always-on (launchd KeepAlive): 
    cp launchd/com.hiretyler.jobbuddy.dashboard.plist ~/Library/LaunchAgents/
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.hiretyler.jobbuddy.dashboard.plist
  7. Open http://localhost:3000 and re-drag the bookmarklet above.

After any server or prompt change, restart with:

launchctl kickstart -k gui/$(id -u)/com.hiretyler.jobbuddy.dashboard

The plist hardcodes the node path (/opt/homebrew/bin/node) and the repo path. If either differs on a fresh machine, edit launchd/com.hiretyler.jobbuddy.dashboard.plist before loading it. Static web/ assets serve fresh on a hard refresh; only server/prompt changes need the restart.
